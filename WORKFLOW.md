# 📋 Batch Validator 标准工作流程

## Excel vs PPT 批量校验 标准执行流程

**触发条件**：用户输入包含「校验Excel」、「验证PPT」、「批量校验」等关键词

---

### **Step 1: 获取文件路径**
```
输入："帮我校验这个Excel和PPT的信息是否一致"
处理：
- 识别校验请求
- 询问用户提供文件路径
- 确认文件存在
```
**输出**：
```
✅ 已识别到批量校验请求
📋 请提供以下文件：
  1. Excel文件（信息列表）
  2. PPT文件（对应截图和文字标注）
```

---

### **Step 2: 提取PPT文字内容**
```
执行：
1. 使用python-pptx读取PPT文件
2. 提取每页的所有文字内容
3. 保存到临时JSON文件
```
**输出**：
```
Step 1: 提取PPT文字内容...
Slide 1: 媒体平台：易车, 见刊日期：2024/11/15...
Slide 2: 媒体平台：凤凰汽车, 见刊日期：2024/11/15...
...
✓ Text content saved to: ppt_text_temp.json
```

---

### **Step 3: 提取PPT截图**
```
执行：
1. 从PPTX文件中提取每页的嵌入图片
2. 找到每页最大的图片（通常是截图）
3. 保存为 slide_001.png, slide_002.png...
```
**输出**：
```
Step 2: 提取PPT幻灯片截图...
PPT has 10 slides
Processing slide 1...
  ✓ Saved: slide_001.png (3071x1919)
Processing slide 2...
  ✓ Saved: slide_002.png (2880x1800)
...
✓ Extracted 10 slide images
```

---

### **Step 4: 解析PPT数据**
```
执行：
1. 从PPT文字中提取结构化信息
2. 使用正则匹配字段：
   - 媒体平台
   - 见刊标题
   - 见刊日期
   - 见刊位置
   - 见刊账号
   - 见刊链接
```
**输出**：
```
Step 3: 解析PPT数据...
解析完成，提取到10页数据
```

---

### **Step 5: 读取Excel数据**
```
执行：
1. 使用openpyxl加载Excel文件
2. 读取表头和数据行
3. 确认数据结构
```
**输出**：
```
Step 5: 读取Excel数据...
  Excel行数: 10
  列名: 序号, 媒体名称/平台, 粉丝量, 发布账号, 发布日期, 刊出位置, 文章标题, 链接, 阅读量
```

---

### **Step 6: 执行7项校验（含智能数据提取）**

对每一行执行：

#### 6.1 基础信息校验（从PPT文字）
```
校验1-发布平台: Excel"易车" vs PPT"易车" → 是
校验2-文章标题: Excel"xxx" vs PPT"xxx" → 是
校验3-发布时间: Excel"2024-11-15" vs PPT"2024/11/15" → 是（规范化后）
校验4-发布账号: Excel"/" vs PPT"汽车新动态" → 否：Excel未记录
校验7-见刊位置: Excel"PC端首页焦点图" vs PPT"PC端首页焦点图" → 是
```

#### 6.2 统计数据校验（3级优先级提取）
```
校验5-粉丝数量:
  [Tier 1] PPT文字 → 未找到
  [Tier 2] 截图识别 → 找到: 1647
  [Tier 3] 网页抓取 → 跳过（已找到）
  结果: Excel"/" vs PPT"1647" → 信息：截图显示1647，Excel未记录

校验6-阅读量:
  [Tier 1] PPT文字 → 未找到
  [Tier 2] 截图识别 → 未找到
  [Tier 3] 网页抓取 → 未找到
  结果: N/A：PPT文字、截图和网页均未找到阅读量
```

**输出**：
```
第1行:
  获取统计数据（3级优先级：PPT文字 → 截图 → 网页）
    ✗ 粉丝数: 未找到
    ✗ 阅读量: 未找到
  ✓ 校验完成

第2行:
  获取统计数据（3级优先级：PPT文字 → 截图 → 网页）
    ✗ 粉丝数: 未找到
    ✗ 阅读量: 未找到
  ✓ 校验完成

第3行:
  获取统计数据（3级优先级：PPT文字 → 截图 → 网页）
    ✗ 粉丝数: 未找到
    ✓ 阅读量: 42 (来源:截图)
  ✓ 校验完成
```

---

### **Step 7: 生成格式化Excel**
```
执行：
1. 在原Excel基础上添加7列校验结果
2. 应用颜色编码：
   - 绿色 = 是（通过）
   - 红色 = 否（失败）
   - 黄色 = N/A或信息
3. 设置列宽为35字符
4. 启用自动换行
5. 保存到原文件同目录
```
**输出**：
```
Step 7: 生成结果文件...
✓ 校验结果已保存：/path/to/原文件名-校验结果.xlsx
```

---

### **Step 8: 汇总报告**
```
执行：
1. 统计总行数
2. 计算全部通过的行数
3. 统计存在问题的行数
```
**输出**：
```
================================================================================
校验汇总
================================================================================
总计: 10 行
全部通过: 8 行
存在问题: 2 行

颜色说明：
  🟢 绿色 = 是（通过）
  🔴 红色 = 否（未通过）
  🟡 黄色 = N/A（不适用）
```

## 异常处理流程

### 🔴 情况1：PPT无文字标注
```
输出：
⚠️  PPT中未找到文字标注
💡 建议：
1. 确认PPT每页是否包含标注信息（媒体平台、见刊标题等）
2. 使用纯截图模式（需手动补充screenshot_stats.json）
是否继续处理？[Y/n]
```

### 🔴 情况2：Excel和PPT行数不匹配
```
输出：
⚠️  警告: 幻灯片数量(8) 与数据行数(10)不匹配
是否继续校验前8行？[Y/n]
```

### 🔴 情况3：网络抓取失败
```
输出：
⚠️  网页抓取失败（部分链接不可访问）
已完成的校验项：
  • 平台、标题、时间、位置 ✓
未完成的校验项：
  • 粉丝数、阅读量 → 标记为N/A
是否继续生成结果？[Y/n]
```

### 🔴 情况4：截图数据文件格式错误
```
输出：
❌ screenshot_stats.json 格式错误
请确认JSON文件格式正确，或跳过截图数据：
1. 修复JSON文件
2. 跳过截图，仅使用PPT文字和网页数据
选择：[1/2]
```

## 高级用法

### 带截图数据的完整校验

```bash
# 1. 生成截图任务列表
uv run scripts/create_screenshot_tasks.py ./slides ppt_text.json screenshot_stats.json

# 2. Claude手动提取截图数据
# 读取screenshot_stats.json，对每个slide：
#   - Read slide_001.png
#   - 提取粉丝数和阅读量
#   - 更新JSON: {"slide_number": 1, "followers": "1234", "views": "5678"}

# 3. 运行完整校验
uv run scripts/validate.py presentation.pptx data.xlsx screenshot_stats.json

# 结果：包含从截图提取的准确粉丝数/阅读量
```

### 仅校验指定项

修改脚本可支持仅校验部分项，如只校验平台和标题：
```python
# 在validate.py中配置
VALIDATION_ITEMS = [1, 2]  # 只校验项1和项2
```

## 性能优化

- ✅ **异步网页抓取**：多个URL并行请求（可选）
- ✅ **缓存机制**：重复URL不重复抓取
- ✅ **批量处理**：一次处理数百行数据无压力
- ✅ **增量更新**：仅更新变化的行（开发中）
